# RandD — 정부 R&D 연구비 과제 관리

정부 R&D(산업부/KEIT·중기부·복지부 등) 연구비 과제를 기획·작성·관리하는 워크스페이스.

## 폴더 구조
`{주체}/{년도}-{담당기관}-{사업명}/` 규칙으로 과제별 보관.
- **주체(주관, "우리쪽") 3축**: `기출노트` · `탱고북` · `엘피케어`
- 담당기관 = 중앙행정기관 약칭(산자부·중기부·복지부·과기정통부 등)
- 과제 폴더 하위: 작업본 + `제출양식/`(빈 양식) · `제출본/`(작성본) · `연구비/` · `공고/` (각 원본 HWP + 텍스트 추출본)

## 진행 중 과제
- **엘피케어 / 2026-산자부-지식서비스산업기술개발** (마감 6/29, 1단계 BM개발 6개월·1억 그랜트형 → 2단계 기술개발 3년)
  - 작업본: `사업계획서_작성.html` — 과제명 "K-성장 클리닉의 글로벌 진출을 위한 AI 성장케어 통합 플랫폼 개발"
  - 이미지 관리: `이미지_프롬프트.html` / 값 입력·체크리스트: `체크리스트.html` (서버 `서버.py`, 저장 `images/`·`fills.json`)
  - 수익모델: **B2B 단일**(SaaS 구독 + 상생형 기술이전), 보호자 기능은 B2B2C로 포함
- 과거(참고): **엘피케어 / 2026-산자부-디자인산업기술개발** (탈락, 인적·기술 자산 공유)

## 작업 방식
- **HWP**: hwp-reader 스킬로 추출(`extract_hwp.py`) → 프로젝트에 원본+txt 보관
- **그림(3종)**: ①AI생성=`이미지_프롬프트.html`에 붙여넣기(Ctrl+V)/드롭→`images/figN.png` 자동저장(`loadFigImages`가 `.fig` 캡션 `[그림N]`에 주입) ②차트·표·로드맵=사업계획서에 **인라인 SVG/표 직접 작도**(subsec, 그림3·4·12·13) ③admin 실화면=캡처(정부 제출용은 **PHI 실명·차트번호·생년월일 마스킹 필수**). 서버: `python 서버.py 8000`
- **값 입력**: `localhost:8000/체크리스트.html`에서 `[확인]` 항목 값 입력 → "사업계획서에 업데이트" → `fills.json` 저장 → 연구계획서가 로드 시 `data-fill` 자리에 주입(미입력=주황 `[확인]`, 입력=파랑 값)
- **연구계획서 작성 원칙**: 연구계획서 격식체(객관 평서/개조식, 칼럼투 지양), 그림은 간결·명확(심사위원 2초 가독)
